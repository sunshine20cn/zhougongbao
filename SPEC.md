# 周公宝 — AI 解梦 H5

## 1. Concept & Vision

**周公宝**是一款面向中老年用户的AI解梦工具。融合传统周公解梦文化与现代AI技术，用最简单的方式——说或写——获得梦境的解读。界面采用水墨中国风，操作极简，让长辈用起来毫无压力。

**slogan**: 梦里乾坤，周公宝来解

## 2. Design Language

### 色彩系统
- 主色（墨绿）: `#2D5A4A`
- 背景色（宣纸米）: `#F5F0E8`
- 强调色（朱砂红）: `#C0392B`
- 文字主色（墨黑）: `#2C2C2C`
- 文字次色（灰色）: `#7A7A7A`
- 边框/分割线（水墨灰）: `#D4CFC7`

### 字体
- 标题：思源宋体 / Noto Serif SC（衬线，传统感）
- 正文：系统默认无衬线（清晰易读）
- 字号：正文≥18px，标题≥24px，按钮≥20px

### 空间系统
- 页面内边距：20px
- 卡片间距：16px
- 按钮高度：≥56px（老年人友好）
- 圆角：8px（柔和不生硬）

### 动效
- 页面切换：淡入淡出 300ms
- 按钮点击：轻微缩放 scale(0.97) + 颜色加深
- 语音波形：呼吸式脉动动画
- 加载状态：旋转的太极图标的而非 spinner

### 视觉元素
- 图标：线性中式图标（如云、月、山水、书卷）
- 背景：淡雅水墨纹理
- 装饰：毛笔笔触分割线、印章点缀

## 4. Features & Interactions

### 核心功能

#### 4.1 AI解梦（语音）
- **交互**: 首页大按钮"按住说话"，说完松开
- **识别引擎**: Web Speech API（浏览器原生，无需授权）
- **显示**: 识别文字实时显示在输入框中
- **提交**: 识别完毕后自动提交，或手动点击"解读"按钮
- **兜底**: 如果浏览器不支持语音，自动降级为纯文本输入

#### 4.2 AI解梦（文本）
- **交互**: 点击"写梦境" → 进入输入页面
- **输入框**: 多行文本，最少10字，最多500字
- **字数统计**: 实时显示已输入字数
- **提交**: 点击"开始解读"按钮

#### 4.3 解梦结果页
- **展示**: AI生成的解读内容，大字排版
- **语音播报**: "朗读结果"按钮，调用 Web Speech Synthesis API
- **保存**: 自动存入历史记录
- **操作**: 重新解梦 / 查看历史

#### 4.4 历史记录
- **存储**: localStorage（无需登录）
- **列表**: 按时间倒序，显示梦境关键词 + 结果摘要 + 时间
- **详情**: 点击进入详情页，可重新查看完整解读
- **删除**: 左滑/长按删除单条记录

### 边界情况
- 语音无输入 → Toast提示"请再说一次"
- 网络异常 → 显示友好提示，缓存内容可查看
- 空输入提交 → 按钮禁用，不触发
- 历史为空 → 显示引导页面"还没有解梦记录"

## 5. Component Inventory

### DreamInput（主语音按钮）
- **默认**: 大圆形按钮，墨绿色，图标+文字
- **录音中**: 红色圆角，脉动动画，"松开解读"文字
- **识别中**: 加载太极旋转动画
- **禁用**: 灰色（浏览器不支持时）

### ResultCard（解读结果卡片）
- 宣纸纹理背景
- 左侧水墨装饰条
- 正文使用衬线字体
- 底部操作按钮行

### HistoryItem（历史记录条目）
- 左滑显示删除按钮
- 顶部：梦境关键词（截断）
- 底部：时间 + "查看详情>"

### BottomNav（底部导航）
- 两个tab，图标+文字
- 当前tab高亮（朱砂红）

### LoadingState
- 太极八卦图旋转
- 文字提示："周公正在解梦..."

## 6. Technical Approach

### 技术栈
- **框架**: Vue 3（Composition API + `<script setup>`）
- **构建**: Vite 5
- **UI库**: Vant 4（移动端优化）
- **样式**: CSS Variables + Scoped CSS（无 Tailwind）
- **路由**: Vue Router 4
- **状态**: Pinia
- **存储**: localStorage（历史记录）

### 语音技术
- **语音识别**: `webkitSpeechRecognition` / `SpeechRecognition`
- **语音播报**: `window.speechSynthesis`（内置，无需API）

### AI对接
- 调用 OpenClaw 内置模型（通过工具接口）
- 无需用户提供 API Key
